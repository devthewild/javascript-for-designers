ECMAScript is JavaScript

CSS와 HTML처럼 자바스크립트도 버전과 표준이 있다.

“ECMA”는 “유럽 컴퓨터 제조사 연합”의 약자

jQuery는 자바스크립트로 만든 것이다.

막 입문한 사람이면 좀 헷갈릴 수 있다.

라이브러리는 코드를 모아둔 것으로, 직접 많은 코드를 짤 필요없이 더 많은 것을 할 수 있게 해주는 것이다.

Objects

모든 것은 객체

객체를 이해하면 자바스크립트를 이해할 것이다. Cody Lindley, 자바스크립트를 깨우치다

객체는 어떻게 생긴 것인가?

객체는 어떤 것들의 모음이고, 어떤 것들은 속성들이라 부른다.

객체를 만들기

Object Literal을 사용해서 객체를 만들 수 있다:

var unicorn = {
  name: "Twilight Sparkle",
  color: "purple",
  wings: true,
  getName: function() {return "Hi, I'm " + unicorn.name;}
};

Object literal은 객체를 만드는데 선호받는 방법이다.

변수 선언하기

객체를 변수처럼 저장할 수도 있다.

var unicorn = {};
unicorn.name = "Rarity";
unicorn.color = "white";
unicorn.wings = false;
unicorn.getName = function() {return "Hi, I'm " + unicorn.name;};

복잡한 객체

많은 종류의 객체들이 있다. 시작해보자.

  • Object 는 보통의 자바스크립트 객체다: 
    var boots = {};
  • Array는 목록을 만드는 좋은 방법이다: 
    var toDo = ['Forge Sword', 'Slay Dragon', 'Ride off into Sunset'];
  • Function 뭔가를 하는 것이다. (그 이상은 곧 다룰 것!)

다양한 많은 데이터를 포함할 수 있기에 복잡하다.

Primitive Values

Primitive Values는 값을 표현할 수 있는 가장 단순한 형태이다.

  • String은 그냥 텍스트: 
    var ballad = "I miss you, but I haven't met you yet.";
  • Boolean은 두가지, true/false 값을 갖는다: 
    var sword = true;
  • Number 수치값 : 
    var castles = 15;

복잡한 객체의 비교

복잡한 객체는 값으로 비교되는게 아니다.

var castles = ["summer", "winter"];
var pegasi = ["Rainbow Dash", "Fluttershy"];

castles === pegasi; // 두 Object들은 수치값이 아니다.

Primitives를 비교하기

Primitives는 객체가 아니다. 값이다.

var castles = 2;
var pegasi = 2;

castles === pegasi; // 수치값 비교하기

왜 ===를 써야하는지 알아보자.

전역 객체: 객체의 신

이 모든 객체들을 품어살피시는 하나의 객체가 있나니: 전역객체.

브라우져에서는 윈도우window가 전역 객체.

또한 Head Object 혹은 Host Object로 불리기도 한다.

window.alert("OMG!")는 묵시적으로 alert("OMG!")와 같다.

jQuery로 객체 다루기

항상 jQuery에 객체를 통과시킨다.!

$.ajax({ // an ajax call

$("input").click(function(){

    "color": "#a52469",

  });
});

Functions

함수도 객체다!

함수는 1등급 객체이며, 무슨 뜻이냐면:

  • 그 자체로생성자가 된다
  • 파라미터로 넘길 수 있다
  • 연산의 반환값이 된다
  • 변수로 할당할 수 있다

메소드

함수가 어떤 객체의 속성이 될 때, 이를 (그 객체의)메소드라 부른다.

unicorn
속성: 속성값:

color purple
wings true
getName function() {
  return "Hi, I'm " + unicorn.name;
}		 
var unicorn = {
  name: "Twilight Sparkle",
  color: "purple",

  getName: function() {return "Hi, I'm " + unicorn.name;}
};

함수 만들기

// 함수 선언
function addStatement(x, y) {
  return x + y; 
}

// 함수 표현식
var addExpression = function(x, y) {
    return x + y;
};
함수의 종류에 대한 자세한 글: 함수선언 vs 함수표현

함수가 하는 일

함수는 객체를 생성하기 위해, 값을 반환하기 위해, 혹은 그냥 코드를 실행하기 위해 쓰일 수 있다.

JavaScript에서의 동사.

document.querySelectorAll("p");

함수 실행하기

함수를 실행하려면 괄호가 필요하다.

$("#thief").hide();

괄호 안에서 쓰인 것은 함수에서 쓸 수 있는 인자가 된다.

function addition(x, y) {
  return x + y; 
}

함수에서의 반환

return은 값이나 객체를 함수에서 반환해준다.

function redHen() {
    return {
        laidEggs: "5 eggs",
        poops: "5 times a day"
    }
    console.log("Plot against the humans."); // return 뒤의 것들은 실행되지 않는다.
}

var henrietta = redHen(); // 함수를 통해 객체를 만들기

또한 함수에서 그 뒤의 부분은 실행되지 않는다.

익명 함수

익명 함수는 일회용 함수같은 것이며 필요할 때 쓰인다.

$("#rebel").click(function() { // 익명 함수
  console.log("The new world order has begun.");
});

Callbacks

콜백은 다른 함수에서 나중에 실행할 수 있도록 넘겨주는 것이다.

$("#goodbye").click(function(){ // 클릭 이벤트 후에 실행
  alert("Holla back, y'all!");
});

setTimeout(function(){ // 3초 후 실행
  alert("Ready for lift off.");
},3000);

인자로 함수를 받거나 함수를 반환하는 것을 고차원 함수(higher-order funciton)이라 한다.

Making Objects with Functions

생성자 함수

같은 종류의 많은 객체를 만들어야할 때 유용하다.

// 함수 표현식으로 생성자를 만들기!
var Unicorn = function(name, color, wings) { 
  this.name = name;
  this.color = color;
  this.wings = wings;
  this.getName = function() {return "Hi, I'm " + this.name;};
};

var twilightSparkle = new Unicorn('Twilight Sparkle', 'purple', true);

twilightSparkle.constructor

변수의 범위

함수 내에서 선언된 변수는 오직 함수 내에서만 접근가능하다.

var cat = "An adorable pet that purrs cutely."; // 전역 cat

animalpedia = function() {
  var cat = "A menace to birds."; // 지역 cat
  console.log(cat);
}

좋아! var를 함수 내부에서 쓰지 않았다면, 변수는 전역 영역에 속한다!

this

함수를 만들면, this가 생긴다.

this는 메소드를 호출한 컨텍스트를 가리키는 변수처럼 쓰인다.

var ourObject = {
  func: function() {
    console.log(this); // 컨텍스트를 가리킨다.
  }  
}

ourObject.func(); // ourObject가 컨텍스트

this와 전역 객체

기본적으로, this는 전역 객체를 가리킨다. (왜냐면 모든 것을 품으니까).

console.log(this);

좀 오해할 수 있는 부분!

var newFunc = ourObject.func; 
newFunc(); // 전역 객체를 컨텍스트로!

The jQuery object, AKA $

기본적으로, jQuery는 “$”를 “jQuery”의 약칭으로 쓴다.

그러니까 $("#id").hide(); 는 jQuery("#id").hide();와 같다.

jQuery는 단지 많은 메소드를 가지고 있는 큰 함수다!

jQuery와 함께 함수 사용하기

항상 jQuery와 함께 함수를 사용할 수 있다.

$("#button1").click(function(){
  var buttonText = $("#button1").val();
  alert(buttonText);
});

$("#button2").hover(function(){
  var buttonText = $("#button2").val();
  alert(buttonText);
});

CodePen에서 실행하기!

반복하지마!

this를 사용하면, 함수를 재사용하기 쉽다.

var logButton = function(){
  var buttonText = $(this).val();

}

$("#button2").hover(logButton);

The DOM API

API란?

Application programming interface (API)는 다른 프로그램들끼리 서로 소통하는데 쓸 수 있는 명령들의 집합이다.

여러분이 사용할 수 있는 많은 API들이 있다.

브라우져 API들로는Web Audio API와 Canvas API같은 것들이 있다.

또 어떤 회사의 데이터를 개발자들에게 제공해주는 트위터의 API같은 것도 있다.

The DOM and jQuery

페이지의 내용을 자바스크립트를 사용해서 다뤄야할 때 DOM(Document Object Model) API를 사용할 것이다.

HTML 트리에 있는 모든 요소(element)들은 JS에서 node라 불리는 것이다.

jQuery에서는 DOM을 다루거나 가져올 수 있는 표준화된 메소드를 제공한다.

전역 객체와 document

브라우져에서 우리의 전역 객체 window 는 항상 document 노드를 갖는다: the HTML document.

앞에서 봤던 document 기억해?

document.querySelectorAll('p'); // JS로 p 가져오기

이벤트와 그 리스너

    Getters and Setters

    Closures

    변수가 함수 안에서 선언될 때, 그 함수나 그 함수가 포함하는 함수에서만 접근할 수 있다.

    클로저는 함수 안에 함수가 중첩될 때 일어난다…

    …그리고 그때 포함하는 함수의 변수를 사용한다!

    A Closure in Action

    function princess() {
  var dragons = ["Great Blue", "Glitterfang", "Fafnir"]; // there are 3 dragons to slay

  return {
    slayDragon: function() {
      return dragons.pop(); //pop gets and removes the last array item
    }
  };
}

var littleGirl = princess(); //littleGirl is now secretly a princess

// Now she can slay dragons no one can see.
littleGirl.slayDragon(); 
littleGirl.slayDragon(); 
littleGirl.slayDragon();

    Recursion

    Recursion is when a function calls itself. See this example from Cody Lindley:

    var countDownFrom = function countDownFrom(num) {
  console.log(num);
  num--;//reduce num by 1
  if (num < 0){return false;} // if num < 0 return function with no recursion
  countDownFrom(num);
};
countDownFrom(5); // kick off the function, which logs separately 5,4,3,2,1,0
